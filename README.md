## Course Structure / Կուրսի կառուցվածք

This repository contains all materials from the 10-month Machine Learning Engineer program at Armenian Code Academy (ACA), including homework solutions, hands-on projects, and structured notes covering all 11 sections of the program.

Այս պահոցը ներառում է Armenian Code Academy (ACA)-ի 10-ամսյա Մեքենայական ուսուցման ինժեների ծրագրի բոլոր նյութերը՝ տնային առաջադրանքների լուծումներ, գործնական նախագծեր և կառուցվածքային նշումներ՝ ընդգրկելով ծրագրի բոլոր 11 բաժինները։

## Naming Convention / Ֆայլերի անվանում

All files follow a consistent naming format for easy navigation and searchability. / Բոլոր ֆայլերն ունեն միասնական անվանման ձևաչափ, որը հեշտացնում է նավիգացիան և որոնումը։

```
prefix_s##_l##_topic_name.extension
```

| Where                                   | Որտեղ                                         |
| --------------------------------------- | --------------------------------------------- |
| `s##` → section number (e.g., s01, s02) | `s##` → բաժնի համարը (օր.` s01, s02`)         |
| `l##` → lecture number (e.g., l01, l03) | `l##` → դասախոսության համարը (օր.` l01, l03`) |

Example / Օրինակ:
`hw_s01_l03_linear_algebra_i.ipynb`

| Prefix / Նախամասնիկ | Type              | Տեսակ                      | Example / Օրինակ                     |
| ------------------- | ----------------- | -------------------------- | ------------------------------------ |
| `hw`                | Homework solution | Տնային առաջադրանքի լուծում | `hw_s01_l03_linear_algebra_i.ipynb`  |
| `lec`               | Lecture note      | Դասախոսության նշում        | `lec_s01_l01_python_fundamentals.md` |
| `note`              | Concept note      | Կոնցեպտի նշում             | `note_gradient_descent.md`           |
| `proj`              | Project file      | Նախագծի ֆայլ               | `proj_capstone_eda_v1.ipynb`         |

- All names are lowercase, separated by underscores / Բոլոր անունները փոքրատառ են և բաժանված են `_`-ով
- Date is tracked by Git history, not the filename / Ամսաթիվը պահվում է Git history-ում, ոչ թե ֆայլի անվան մեջ
- Versions are marked with `_v1`, `_v2` suffixes for projects / Տարբերակները նշվում են `_v1`, `_v2` վերջածանցներով նախագծերի համար

**Language Policy / Լեզվական քաղաքականություն**

All code, comments, variable names, notes, and project deliverables are written entirely in English — the standard language of the global Machine Learning and Software Engineering industry. Personal notes, conceptual explanations, and learning materials are written primarily in Armenian. This intentional choice supports deeper conceptual understanding, stronger long-term retention, and more effective mastery of complex technical topics. English summaries or translations are provided where appropriate to ensure full international accessibility.

Բոլոր կոդերը, մեկնաբանությունները, փոփոխականների անունները, նշումները և նախագծային արդյունքները գրված են ամբողջությամբ անգլերենով՝ որպես մեքենայական ուսուցման և ծրագրային ճարտարագիտության ոլորտի համաշխարհային ստանդարտ լեզու։ Անձնական նշումները, կոնցեպտուալ բացատրությունները և ուսումնական նյութերը հիմնականում գրված են հայերեն՝ ավելի խորքային հասկացողության, երկարաժամկետ հիշողության ամրապնդման և բարդ տեխնիկական թեմաների արդյունավետ յուրացման նպատակով։ Անհրաժեշտ դեպքերում տրամադրվում են անգլերեն ամփոփումներ կամ թարգմանություններ՝ միջազգային լիարժեք հասանելիությունն ապահովելու համար։

**Commit Message Convention / Commit հաղորդագրությունների ձևաչափային կանոններ**

| Type        | Description                                                         | Նկարագրություն                                                           |
| ----------- | ------------------------------------------------------------------- | ------------------------------------------------------------------------ |
| `feat(s##)` | New feature, homework solution, or note added in a specific section | Նոր ֆունկցիա, տնային առաջադրանք կամ նշում՝ կոնկրետ բաժնում               |
| `fix(s##)`  | Bug fix or correction in code, notes, or solutions within a section | Սխալի ուղղում կոդում, նշումներում կամ լուծումներում                      |
| `docs`      | Documentation updates (README, comments, explanations)              | Փաստաթղթավորման թարմացում (README, մեկնաբանություններ, բացատրություններ) |
| `refactor`  | Code or structure changes without modifying core content or meaning | Կոդի կամ կառուցվածքի փոփոխություն՝ առանց բովանդակության փոփոխման         |
| `wip(s##)`  | Work in progress — incomplete implementation or ongoing work        | Աշխատանք ընթացքի մեջ՝ անավարտ կամ շարունակվող փոփոխություն               |
